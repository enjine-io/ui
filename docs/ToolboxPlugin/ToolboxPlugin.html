Toolbox Plugin


Toolbox Plugin is just a DroidScript Hybrid plugin with additional files to support the Layout Extension drag and drop feature.

Plugin Directory


These are the files needed in order to add your custom component or Hybrid plugin into the Layout Extension or WYSIWYG Editor.

EXAMPLE - PLUGIN DIRECTORY

   - MyPlugin.inc
   - MyPlugin.html
   - MyPlugin.js
   - tools.json

MyPlugin.inc


This file is your main plugin file where you are going to write your code for your custom controls. This will be save in DroidScript Plugins folder.

EXAMPLE - MYPLUGIN.INC

   var MyPlugin = {}

   MyPlugin.addButton = function(parent, text, options, width, height) {
       return new MyPlugin.MyButton( parent, text, options, width, height )
   }

   MyPlugin.MyButton = class extends ui.Control
   {
       constructor(parent, text, options, width, height)
       {
           super(parent, width, height, options, "MyPluginButton")
           this._onTouch = null
           this._el = document.createElement("button")
           this._el.innerText = text || "My Button"
           this._el.style.width = "100%"
           this._el.style.height = "100%"
           this._el.onclick = this._touch.bind(this)
           this._div.appendChild( this._el )
       }
       
       _touch() {
           if(this._onTouch) this._onTouch()
       }
       
       // sample setter property
       set text( text ) { this._el.innerText = text }
       
       // sample setter method
       setSize(size)
       {
           size = size ? size.toLowerCase().trim() : "medium"
           if(size == "small") this._div.style.transform = "scale(0.8,0.8)"
           else if(size == "medium") this._div.style.transform = "scale(1,1)"
           else this._div.style.transform = "scale(1.5,1.5)"
       }
       
       // sample event
       setOnTouch(callback) { this._onTouch = callback }
   }

Looking at the example above, your custom control should inherit the ui.Control class.

2. Control class

It doesn't matter what is the order of your parameters text, width, height, options and so on, as long as the first parameter is a ui.Layout component which is the parent layout where to add your custom control.


The super method should have the correct order and exact type of arguments.

ParametersTypeDescription
parent Layoutui.Layout component.
width NumberFraction of the parent width. Value ranges from 0 to 1
height NumberFraction of the parent height. Value ranges from 0 to 1
options StringA comma separated string of options Pass null if you don't have it.
type StringYour control custom type. Should be single word. e.g. Button, Mapview, Card, Radio and so on.

tools.json


This file holds the json for adding your controls in the component list and their corresponding properties and events.
Your tools.json file should look like this

EXAMPLE - TOOLS.JSON

   {
       "global": "MyPlugin",
       "name": "MyPlugin",
       "version": "1.0.0",
       "author": "John Doe",
       "components": {
           "Button": {
               "icon": "favorite",
               "methodName": "addButton",
               "params": ["My Button", "", 0.5, 0.1],
               "events": {
                   "setOnTouch": [ "text" ]
               },
               "setters": [
                   ["text", "", true, "", "p2", "Sets the button text"],
                   ["setSize", "Small,Medium,Large", false, "method", "p3", "Sets the button size"]
               ]
           }
       }
   }
1. General informations

ParametersTypeDescription
global StringThis is the name of your plugin object that is accessible globally. If your control can be called using MyPlugin.addButton(), your global namespace is MyPlugin. In some instances where your control is a method of ui namespace e.g. ui.addCustomButton your global namespace will be ui. If you don't pass a name, it will be default to your plugin name.
name StringThe name of your plugin.
version StringThe current version of your plugin.
author StringThough not required, if not provided it will be default to Unknown
components ObjectThis object is composed of components you want to add into the WYSIWYG Editor.
2. Component Properties

Below are the description of the properties you need to provide in each component.

ParametersTypeDescription
icon StringThe custom material icon of the component. Default is add icon.
methodName StringThe callname of the method. e.g. addButton or addLayout
params ArrayAn array of the default values of the method arguments. See the Param Array below for more details.
setters ArrayAn array of methods and available values which will be displayed in the Properties tab in the right panel. See Setters Array below for more details.
events ObjectAn object container the supported events and their returned values. See Event Object below for more details.
2.1 Params Array

The params is an array of all the method parameters excluding the first parameter which is the parent. These will be use as the default values for the method arguments when the component is selected on the left panel of the Layout Extension Editor.
This array is of the form

[arg2, arg3, arg4, arg5, ...]

arg2 : The value of the second parameter
arg3 : The value of the third parameter
arg4, arg5 and so on.


Example

MyPlugin.addButton(parent, text, options, width, height)

The params should be like these

["My Button", null, 0.5, 0.1 ]

Where
My Button is the default value of button.
null options value is default ot null.
0.5 is the default width.
0.1 is the default height.

2.2 Setters Array

These properties and methods will be displayed at the right panel in the Properties tab.


The setter is an array where each element is an array of the form

[name, defaultValue, editable, options, param, hint]
ParametersTypeDescription
name StringThe name of the property or method. e.g. text or setSize. For method pass method in the options argument.
defaultValue StringThe default value of the TextField or Select. For select, pass a comma separated choices. You can follow the format on adding list items on Select Component. See the UI Select documentation.
editable BooleanIf true it will render a TextField component where you can type and edit the value. If false it will render a Select component where you must provide a comma separated choices as the defaultValue stated above.
options StringA comma separated options for the TextField or Select component. With additional options. If you have a setter method that accepts more than one arguments (e.g. setMargins(left,top,right,bottom)) pass a list options so that a comma separated values will be pass as arguments for left top right and bottom. If you have a setter method that accepts color, you can pass a color option to add color picker as an end adornment. If you have a setter method that accepts a file path, pass a path option. Pass a method option if this is a method. By defaut it will be treated as property.
param StringThough not required, this will tell the Editor to save the entered value as the value of the corresponding argument on control initialization. Example, if a control is created using MyPlugin.addButton(lay, text, width, height) and you have a text setter property, your setter array should be ["text", "", true, "", "p2"]. This means that the value of text TextField when edited will be use as the 2nd parameter text when the control is created. This will reduce the time of rendering because it will not set the text property to have the same text displayed. Please note that values should start at p2 as the p1 is the parent layout, then p3 and so on.
hint StringThis will be displayed when the name of the property or method is click to provide some hint.

Example 1

["text", "", true, "Text", "p2", "Sets the text."]

text : This means that the property name is text.
"" : There is no default value.
true : This is a textfield and should be editable.
"Text" : The textfield should be of input type Text. See TextField inputType property for available values.
p2 : The entered value should be the value of param2 say MyPlugin.addButton(lay, text, options, width, height)
"Sets the text." : Hint text.


Example 2

["backColor", "", true, "Color", "", "Sets the background-color."]

backColor : This means that the property name is backColor.
"" : There is no default value.
true : This is a textfield and should be editable.
Color : The textfield shall have an additional color palette icon where to choose color.
"" : No param is associated with it.
--hint-- : Hint text


Example 3

["setSize", "Small,Medium,Large", false, "Method", "", "Sets the size"]

setSize : This means that the method name is setSize.
Small,Medium,Large : Available values where to choose from.
false : This is a select component.
Method : This is a method and not a setter property.
"" : No param is associated with it.
--hint-- : Hint text

2.3 Event Object

These events object will be displayed as the options when you defined a callback in the Events tab at the right panel.
The property name or key is the name of the callback and the value of each property is an array of values that will be passed to the callback if any.


Example setOnTouch: ["text", "index"]


This means that the callback will receive a text and index as arguments. Pass empty array [] if you have none to pass to the callback.
