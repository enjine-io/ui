Page

Load the pages created by the Layout Extension or WYSIWYG Editor. Call this method like this

ui.load()

By default, this will create an Absolute main layout where all the pages will be rendered. In order to render a page
on a specific layout which you have defined in your app, pass a parents object of the form

{ page1: lay1, page2: lay2 }

This will render page1 on layout lay1 and page2 will be rendered on layout lay2.

ParametersTypeDescription
parents ObjectA key value pair where the key is the name of the page and the value is a Layout component.

After loading all the pages, you can access each page object on the ui.pages property using its pageName. The following example assumes
that we have two pages namely page1 and page2 and in order to access page1 object, call it like this

page1 = ui.pages[ "page1" ]

Similarly, for page2

secondPage = ui.pages[ "page2" ]

METHODS

These are the methods available for each page object.

load

Load the page on a given layout.

page.load()
ParametersTypeDescription
parent LayoutThe layout where to render the page.

Please note that these method is already executed when calling ui.load(). Calling it without passing a Layout argument or
without changing its parent layout is just a waste of time. Refrain from calling page.load() unless you want to change the parent of a page.

animate

Animate the page. You can pass either entrance or exit animation for the anim argument.

ParametersTypeDescription
anim StringThe animation for the page. Refer to UI Components for available values.
duration NumberThe animation duration in milliseconds.
exit BooleanSome aimation don't have the word out in it so pass and set this argument to specify whether the animation is for exit or not.
show

Shows the page.

page.show()
hide

Hides the page.

page.hide()

do nothing


To navigate into the previous page use ui.goBack(). This will hide the last page in the pageStack tree.
